Main Circuit

Kuman UNO R3

Used as the main processing unit for the system, UNO R3 executes the program(s) developed by Arduino Software (ARDUINO 1.8.5) IDE.

Power Supply

The power supply supplies the UNO R3 boards, Fingerprint Scanner and Stepper Motors with 5 DC volts.

Breadboard Module Adapter Shield

Used as a precaution to avoid overheating of the UNO R3 boards.

Power Splitter

Used to split incoming 9V into the UNO R3 board and Breadboard Module Adapter Shield.

Stepper Motor Driver Boards

Connected to the two Stepper Motors, these Driver Boards control the magnitude and direction of current flow into the Stepper Motors.

Potentiometer

By turning the shaft og the potentiometer, we can change the amount of resistance sent to the LCD Screen

LCD + FPS Compartment

LCD Screen

With utilizing LiquidCrystal Library LCD Module is used to display the instructions of the system to the user. The display is controlled by using 6 data pins. 

                 LiquidCrystal lcd(RS, E, D4, D5, D6, D7)

Where the RS pin indicates the number of UNO R3 pin that is connected to the RS pin on the LCD (which is pin 7, in our case).

The E pin referes to the enable pin, which is the number of UNO R3 pin that is connected to the enable (E) pin on the LCD (which is pin 8, in our case).

Pins D4, D5, D6, D7 the numbers of the UNO R3 pins that are connected to the corresponding data pins on the LCD display.

Fingerprint Scanner

Most important component of the Fingerprint Garage Door, this piece of hardware allows the user to interact with the system using their fingerprint.

Circuit in Detail

Fingerprint Scanner - TTL GT-521F32

For our project, we have utilized the Fingerprint Scanner - TTL GT-521F32. This hardware includes an optical sensor for reading in fingerprints and a processing IC with built-in fingerprint algorithms. Being inexpensive, it can remarkably hold upto 200 unique fingerprints. The image to the left shows the FPS's optical sensing area, where the device will be able to scan fingerprints.


Enrolling

To enroll a fingerprint to the module, the user needs to enroll the finger three times for each ID before the scanner can save it as a template in the authorized fingerprints database. Once the white LED lights up, the scanner is ready to read fingerprints. If the scanner is unable to recognize a fingerprint or detect a finger that was placed on the scanner, it will terminate the enrollment and the LCD screen will print "Access Denied". Always make sure there is suffcient contact with the scanner and the finger is placed in the same position during enrollment.


Identifying

To test the fingerprint after enrolling, the user will need to press the red pushbutton for greater than 3 seconds and less than 10 seconds. The LCD screen will print "Place the finger to enroll on pad”. The code portion of the system will check through the database of authorized fingerprints to see if it can recognize the fingerprint against the saved fingerprint templates. If a match is found successfully, it will respond with opening the garage panel of the garage model.

Note

This system assumes that the user is using the latest version of the Arduino IDE as well as the Hawley's FPS_GT511C3 Arduino Library. Also, the FPS_GT511C3 library may not work with all microcontrollers using the Arduino IDE. As you move away from ATmega328P family, you may need to modify the code or port the library over to get it working.
